# Заметки
* Принцип единственной ответственности (Single Responsibility)
* Принцип наименьшего знания (Law of Demeter)
* Функция `remember{}` позволяет сохранить значения между рекомпозициями
* Ленивые списки отображают данные по мере необходимости
* Правила разработки компонентов:
  Компоненты должны принимать в качестве параметров:
    - Modifier
    - Данные для отображения
    - Callbacks
* SideEffect - callback будет вызван при каждой рекомпозиции
* LaunchedEffect - callback будет вызван после рекомпозиции, если значение ключа поменялось
